# ペルソナ & User Journey Map

Tech Feed のターゲットユーザーを定義し、利用体験を可視化したドキュメントです。

---

## ペルソナ

### ペルソナ1: 田中 健太（通勤エンジニア）

**基本情報**:

- 年齢: 29歳
- 職業: フロントエンドエンジニア（5年）
- 勤務形態: 週5日出社
- 通勤時間: 片道30分

**利用シーン**:

- 朝の通勤電車内（8:00-8:30）
- 昼休憩（12:30-12:45）
- 帰宅時の電車内（19:00-19:30）

**ゴール**:

- 朝の5分で重要記事をキャッチアップ
- 効率的な情報収集
- スキルアップ

**ペインポイント**:

- 時間が限られている（5-15分）
- 同じ記事を何度も見る
- フィルター選択が面倒
- 既読管理がない

---

### ペルソナ2: 佐藤 美咲（育児エンジニア）

**基本情報**:

- 年齢: 32歳
- 職業: フルスタックエンジニア（8年）
- 勤務形態: フルリモート
- 家族構成: 既婚、4歳の子供

**利用シーン**:

- 朝活カフェ（8:00-8:30）
- 子供の昼寝中（14:00-14:30）
- 夜、子供が寝た後（21:00-22:00）

**ゴール**:

- スキマ時間での効率的学習
- キャリアアップ（デザインエンジニアへ転向）
- 学習履歴の記録

**ペインポイント**:

- 時間制約が厳しい（5-10分単位）
- 読了時間を事前に知りたい
- 学習記録でモチベーション維持
- 孤独感（チーム外での学習）

## User Journey Map

### ペルソナ1（田中健太）: 朝の通勤

| フェーズ | 行動 | 思考 | 感情 | ペインポイント | 改善機会 |
| --- | --- | --- | --- | --- | --- |
| **準備** | 通勤電車に乗る | 「今日は何を読もう」 | 期待 | - | - |
| **検索** | Tech Feedを開く | 「タグを選ばないと...」 | 中立 | フィルター選択に時間 | 前回設定を復元 |
| **閲覧** | 記事一覧を見る | 「あ、これ昨日読んだ」 | イライラ | 既読管理なし | 既読記事をグレーアウト |
| **選択** | 記事をタップ | 「面白そう、後で読もう」 | 興味 | ブックマーク機能なし | ワンタップでブックマーク |
| **中断** | 駅に到着 | 「結局1本しか読めなかった」 | 不満 | 時間の無駄 | 効率化で2-3本読める |

---

### ペルソナ2（佐藤美咲）: 朝活カフェ

| フェーズ | 行動 | 思考 | 感情 | ペインポイント | 改善機会 |
| --- | --- | --- | --- | --- | --- |
| **準備** | カフェに到着 | 「30分しかない」 | 集中 | - | - |
| **計画** | 何を読むか考える | 「読了時間が分からない」 | 不安 | 時間見積もりなし | 読了時間表示 |
| **閲覧** | 記事を読む | 「これ面白い！」 | 満足 | - | - |
| **記録** | 読み終わる | 「今週何本読んだっけ？」 | 疑問 | 学習記録なし | ダッシュボードで可視化 |
| **終了** | 子供を迎えに行く | 「モチベーション保てるかな」 | 不安 | 進捗が見えない | 週次レポート |

## 実装への反映

| ペインポイント | 対応機能 | ステータス |
| --- | --- | --- |
| フィルター選択が面倒 | カテゴリ選択機能（LocalStorage保存） | 実装済み |
| 時間が限られている | 注目記事（PICK UP!）で重要記事を優先表示 | 実装済み |
| 複数ソースを巡回する手間 | Qiita / Hacker News 統合表示 | 実装済み |
| 英語記事が読みづらい | Hacker News 記事の自動翻訳 | 実装済み |
| ブックマーク機能なし | お気に入り機能（LocalStorage保存） | 実装済み |
| 学習記録がない | ダッシュボード（UI実装済み、データはモック） | 一部実装 |
| 既読管理がない | 既読記事のグレーアウト | 未実装 |
| 読了時間が分からない | 読了時間の表示 | 未実装 |
