# News Curator - 開発ロードマップ

現在の実装状況と今後の開発計画をまとめています。

## 📌 現在のステータス (Current Status)

- **フェーズ**: ベータ版 (v0.1.0)
- **実装済み主要機能**:
  - Qiita API v2 連携による記事取得
  - タグによる記事フィルタリング
  - モバイル対応レスポンシブデザイン
  - 記事検索機能 (クライアントサイド)
  - お気に入り機能 (一時保存)

---

## 🗺️ ロードマップ

### 🚀 短期計画 (Short-term)
**目標**: 複数の技術情報ソースへの対応とユーザー体験の向上

- [ ] **ソース拡張 (Zenn対応)**
  - Zenn RSSフィードからのデータ取得
  - APIルートでのデータ統合
- [ ] **ソース拡張 (Hacker News対応)**
  - Hacker News APIの統合
  - 英語記事のフィルタリングオプション
- [ ] **UIの改善**
  - ダークモードの実装
  - URLクエリパラメータによるフィルタ状態の永続化 (共有可能にする)
  - ローディング状態のスケルトン表示化

### 🛠 中期計画 (Mid-term)
**目標**: パーソナライズ化とパフォーマンス最適化

- [ ] **お気に入り機能の強化**
  - LocalStorage / IndexedDB を用いた永続化
  - 「後で読む」リストの管理画面作成
- [ ] **高度な検索・フィルタ**
  - 公開日でのソート・期間絞り込み
  - 人気順 (いいね数/ストック数) でのソート
- [ ] **パフォーマンス**
  - 画像の最適化 (Next/Image)
  - 仮想スクロール (Infinite Scroll) の導入による大量記事の表示対応

### 🔮 長期計画 (Long-term)
**目標**: ユーザーアカウントとコミュニティ機能

- [ ] **ユーザー認証**
  - Supabase / Clerk 等を使った認証基盤
  - 端末・ブラウザ間での設定同期
- [ ] **カスタムフィード**
  - 独自のRSSフィード登録機能
  - マイタグセットの保存
- [ ] **PWA (Progressive Web App) 化**
  - オフライン閲覧
  - プッシュ通知 (新着記事)
