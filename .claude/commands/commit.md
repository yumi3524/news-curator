# /commit

コミットメッセージを作成する

## 手順

1. `git status` と `git diff --staged` で変更内容を確認
2. 変更内容を分析し、Conventional Commits形式でメッセージを作成
3. メッセージをユーザーに提示（コミットは実行しない）

## 出力形式

```
<type>: <簡潔な説明>

- <変更点1>
- <変更点2>
- <変更点3>
```

## Type一覧

| Type | 用途 |
|------|------|
| feat | 新機能追加 |
| fix | バグ修正 |
| docs | ドキュメント更新 |
| refactor | リファクタリング |
| style | コードフォーマット |
| test | テスト追加・修正 |
| chore | ビルド・設定変更 |

## ルール

- 日本語で記述
- 1行目は50文字以内
- 本文は箇条書きで変更内容を列挙
- コミットは実行せず、メッセージのみ提示
